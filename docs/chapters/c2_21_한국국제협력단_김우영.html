01모델 선택 이유

한국국제협력단(KOICA)은 70여 개 협력국에서 무상원조 사업을 수행하는 국제개발협력 전문기관으로, 영어·한국어를 비롯해 베트남어·캄보디아어·라오어·아랍어·스페인어 등 다양한 협력국 언어가 일상 업무에서 사용된다. Qwen2.5-3B-Instruct는 단일 모델에서 29개 이상의 언어를 지원하면서도 3B 규모로 온프레미스(개인 PC·소형 GPU 서버) 환경에서 무리 없이 구동 가능하며, 32K 토큰의 충분한 컨텍스트 길이를 제공해 사업 보고서 요약·다국어 회의록 정리·협력국 정책 문서 질의응답 등 KOICA의 다국적 행정 환경에 폭넓게 활용할 수 있다고 판단하여 선택했다. 또한 Qwen2.5 시리즈는 동일 규모 대비 한국어·영어 벤치마크에서 경쟁 모델(Llama-3.2-3B, Gemma-2-2B 등) 대비 우수한 성능을 보고하고 있어 PoC 후보로 적합하다.

03학습 데이터와 튜닝 방식

[사전학습] Qwen2.5 기술 보고서(arXiv:2412.15115)에 따르면 Qwen2.5 시리즈는 이전 세대 대비 7조 토큰에서 18조(18T) 토큰으로 확장된 고품질 다국어 코퍼스로 사전학습되었다. 코딩·수학 영역은 별도의 도메인 전문 모델을 활용해 데이터 품질을 강화했다고 보고된다. 다만 정확한 데이터 출처·라이선스·필터링 절차는 모델 카드에 모두 공개되어 있지는 않다. [후처리] Instruct 모델은 사전학습 모델 위에 100만 건 이상의 샘플을 활용한 정교한 SFT(Supervised Fine-Tuning)와 다단계 강화학습(multistage reinforcement learning)을 적용해 명령 수행, 8K 이상 장문 생성, JSON 등 구조화된 출력 능력을 강화했다(기술 보고서 초록 표현). 다만 3B-Instruct 변종에 RLHF·DPO·GRPO 등 세부 단계가 어느 수준까지 적용되었는지는 공개 범위 내에서 명시적으로 확인되지 않는다. [언어] 다국어 학습으로 한국어·영어·중국어·일본어·베트남어·아랍어·스페인어 등 29개 이상의 언어를 직접 지원한다.

04파라미터 및 구조적 특징

ArchitectureQwen2 (Transformer + RoPE + SwiGLU + RMSNorm + GQA + tied word embeddings, bfloat16)
Context Length32,768 토큰 (생성 길이는 권장 8,192 토큰)
Hidden Size2,048 (intermediate_size 11,008)
Layers36
Attention HeadsQ 헤드 16 / KV 헤드 2 (Grouped Query Attention, KV 캐시 절감)
TokenizerBPE 기반 Qwen Tokenizer · vocab size 151,936 (다국어·코드 혼합 토큰화 최적화)

05모델 파일 구성

• config.json — 모델 아키텍처와 하이퍼파라미터(hidden_size 2048, num_hidden_layers 36, num_attention_heads 16, num_key_value_heads 2, vocab_size 151,936, max_position_embeddings 32,768 등)를 정의. transformers 라이브러리가 모델을 로드할 때 가장 먼저 참조하는 파일이다. • model.safetensors — 모델 가중치 본체. bfloat16 정밀도, Safetensors 포맷이라 pickle 기반 .bin 대비 보안성과 로딩 속도가 우수하다. 3B 모델 기준 약 6GB. • tokenizer.json / tokenizer_config.json / vocab.json / merges.txt — Qwen BPE 토크나이저 정의. 다국어·코드 혼합 처리를 위한 어휘와 병합 규칙이 담겨 있다. • generation_config.json — 추론 시 기본 생성 파라미터(temperature, top_p, top_k, repetition_penalty, max_new_tokens 등) 디폴트 값. • special_tokens_map.json / added_tokens.json — <|im_start|>, <|im_end|> 등 채팅 포맷 제어용 특수 토큰 매핑. apply_chat_template 함수가 이 정보를 기반으로 시스템·사용자·어시스턴트 턴을 구분한다. • LICENSE / README.md — 라이선스 본문과 모델 카드(사용법·벤치마크·제한사항 등).

06활용 가능 업무

KOICA 업무 환경에서 검토 가능한 활용 시나리오는 다음과 같다. ① 다국어 사업 보고서 요약 — 협력국 사무소가 제출한 영문·현지어 분기보고서·결과보고서를 한국어로 자동 요약하여 본부 사업부서가 빠르게 핵심을 파악. ② 다국적 회의록 정리 — Zoom·Teams 화상회의의 STT 전사 결과를 입력받아 결정사항·후속조치(Action Item)·담당자별 To-Do로 구조화된 회의록 자동 생성. ③ 사내 RAG 챗봇 — KOICA 사업관리 매뉴얼·OECD DAC 가이드라인·국별 협력전략(CPS) 문서를 벡터 DB로 인덱싱한 뒤, 신규 직원·PMC가 자연어로 질의응답. ④ 사업 기획 초안 작성 — 사업 발굴 단계에서 PCP(Project Concept Paper)·사업기획서 초안을 협력국 배경 정보 기반으로 1차 작성 보조. ⑤ 다국어 커뮤니케이션 — 협력국 카운터파트(영문·스페인어·아랍어 등)와의 이메일·공문 초안을 한국어 키포인트만 입력하면 자동 작성. ⑥ 평가보고서 일관성 점검 — 사후평가 보고서의 OECD DAC 5대 평가기준(적절성·효율성·효과성·영향력·지속가능성) 누락 여부 자동 점검. 모든 시나리오는 내부 직원 보조 도구(Copilot 형태)로 우선 적용하는 것이 안전하다.

07한계와 주의사항

① 라이선스 제한 — Qwen2.5-3B는 Apache-2.0이 아닌 Qwen Research License로, 연구 목적은 자유로우나 외부 서비스·상업적 활용 시 별도 약관 검토와 Alibaba 측 동의가 필요할 수 있다. 정부·공공기관 도입 시 법무·계약 단계에서 라이선스 적합성 사전 검토가 필수다. (참고: 동일 시리즈의 0.5B/1.5B/7B/14B/32B 모델은 Apache-2.0이라 라이선스 부담이 더 적다.) ② 한국어 표현력 한계 — 다국어 모델 특성상 한국어 전용 모델(EXAONE-3.5-2.4B, Polyglot-Ko, KULLM 등) 대비 공문서 어투·존댓말·고유명사·법령 용어에서 미세한 부자연스러움이 나타날 수 있다. ③ 환각(Hallucination) — 3B 규모의 한계로 통계 수치·법령 조항·인명·사업명 등 사실관계가 중요한 항목에서 오류 가능성이 높다. 반드시 RAG(Retrieval-Augmented Generation) 또는 사람의 검증 절차와 병행해야 한다. ④ 보안·개인정보 — 협력국 사업 관련 문서에는 수원국 정부·수혜자 개인정보, 미공개 사업비 정보가 포함될 수 있어 외부 클라우드 추론 API 사용은 부적절하다. 온프레미스 또는 폐쇄망 GPU 환경 운영을 권장한다. ⑤ 컨텍스트 품질 저하 — 32K 토큰을 지원하지만 후반부로 갈수록 정보 활용도가 떨어지는 'lost in the middle' 현상이 보고되어, 매우 긴 문서는 청크 분할·요약 단계 도입이 필요하다. ⑥ 평가 데이터 편향 — 공개 벤치마크 점수가 실제 KOICA 업무 도메인(국제개발협력 전문 용어)과 일치하지 않을 수 있어, 자체 도메인 평가셋 구축이 별도로 요구된다.

08종합 의견

Qwen2.5-3B-Instruct는 KOICA의 다국어 환경과 온프레미스 운용 제약을 동시에 고려할 때 PoC 단계에서 우선 검토할 만한 후보이며, 특히 다국어 요약·번역·내부 RAG 챗봇 영역에서 강점이 기대된다. 다만 본격 도입 전 다음 4가지가 선행되어야 한다고 판단한다. (1) Qwen Research License의 공공기관 적용 가능성에 대한 법무 검토, 또는 대안으로 Apache-2.0 라이선스인 Qwen2.5-7B/1.5B·EXAONE-3.5-2.4B·Llama-3.2-3B 등과의 라이선스·성능 비교. (2) KOICA 도메인 평가셋 구축 — 실제 사업 보고서·CPS·평가보고서 100건 내외로 요약·QA·번역 정량 평가셋을 만들고, BLEU/ROUGE/한국어 LLM-as-a-Judge 점수를 측정. (3) 협력국 주요 언어(베트남·캄보디아·라오·아랍·스페인) 별도 정성 평가 — 단순 영-한 외에 동남아·중동·중남미 언어 품질 확인이 KOICA 업무에서 결정적이다. (4) 한국어 전용 SLM과의 A/B 비교 PoC — 같은 프롬프트·평가셋으로 EXAONE-3.5-2.4B-Instruct 등과 직접 비교하여 다국어 vs 한국어 특화의 트레이드오프를 정량화. 결론적으로 대외 공개 서비스보다는 내부 직원 Copilot(요약·번역·문서 검색)부터 단계적으로 적용하는 것이 안전하며, 약 1~2개월의 PoC 후 본격 도입 여부를 결정하는 로드맵을 제안한다.
