01모델 선택 이유

공공기관 업무 활용 관점에서 다음 세 가지 이유로 Qwen2.5-7B-Instruct를 선택하였다. 첫째, Apache-2.0 라이선스로 상업적 사용·수정·재배포가 모두 허용되어 도청 및 산하 기관 도입 시 라이선스 검토 부담이 가장 낮다. 둘째, 한국어를 포함한 29개 이상 언어를 공식 지원하며 동급(7B) 오픈 모델 중 한국어 instruction following 성능이 최상위권으로 보고되어 민원 응대·문서 처리에 즉시 활용 가능한 수준이다. 셋째, 학습 32K · YaRN 적용 시 최대 128K 토큰의 긴 컨텍스트를 지원해 법령·조례·내부 보고서 등 장문 입력이 잦은 공공 업무에 적합하다. 또한 7.6B 파라미터 규모로 단일 GPU(A100 40GB, RTX 4090 24GB)에서 FP16 추론이 가능하고, 4bit 양자화(GGUF·AWQ) 시 일반 사무용 워크스테이션(VRAM 8GB 이상)에서도 운용 가능해 도입 비용 측면의 가성비도 우수하다.

03학습 데이터와 튜닝 방식

사전학습(Pre-training) 단계에서 약 18조(18T) 토큰 규모의 멀티링구얼 웹 텍스트, 코드, 수학·과학 문헌, 도메인 전문 텍스트로 학습되었다. 한국어를 포함한 29개 이상 언어 데이터가 포함되어 있으나 정확한 언어별 비중·소스 도메인은 공개 범위 내에서는 확인이 어렵다.

후처리(Post-training)는 3단계로 구성된다. ① Supervised Fine-Tuning(SFT): 100만 건 이상의 고품질 instruction-response 페어(인간·합성 데이터 혼합)로 지시 따르기 능력을 학습. ② Offline RL — Direct Preference Optimization(DPO): 인간 선호 데이터로 응답 품질·안전성을 정렬. ③ Online RL — Group Relative Policy Optimization(GRPO): 동일 프롬프트에 대한 다중 응답 비교 보상으로 정책을 추가 정렬.

주요 강화 영역은 코딩(Qwen2.5-Coder 데이터 일부 활용), 수학(Qwen2.5-Math 일부), 다국어 instruction following, 8K+ 장문 생성, JSON 등 구조화 출력이다. LoRA·QLoRA는 공식 배포본에는 사용되지 않았으며(전체 파라미터 학습 기반), 사용자가 추가 도메인 fine-tuning 시 권장되는 경량화 방식이다.

04파라미터 및 구조적 특징

ArchitectureQwen2 (Decoder-only Transformer · RMSNorm · SwiGLU · RoPE · GQA)
Context Length학습 32,768 토큰 / YaRN 적용 시 최대 131,072 토큰 (출력 최대 8,192)
Hidden Size3,584 (intermediate_size 18,944)
Layers28 layers
Attention HeadsQuery Heads 28 / Key-Value Heads 4 — Grouped Query Attention 7:1 (KV 캐시 메모리 절감)
TokenizerByte-level BPE (BBPE) · vocab 151,643 + 특수 토큰 22 · tiktoken 호환

05모델 파일 구성

Hugging Face 저장소(Qwen/Qwen2.5-7B-Instruct)에는 다음과 같은 파일이 포함되어 있다.

● config.json — 아키텍처(Qwen2ForCausalLM), hidden_size·num_hidden_layers·num_attention_heads·num_key_value_heads·max_position_embeddings·rope_theta·intermediate_size 등 모델 구조 정의
● generation_config.json — 기본 생성 파라미터(temperature·top_p·top_k·repetition_penalty·eos_token_id·max_new_tokens) 정의
● tokenizer.json / tokenizer_config.json — BBPE 토크나이저 어휘·병합 규칙·특수 토큰 매핑·chat_template(ChatML 형식) 정의
● vocab.json + merges.txt — BBPE 어휘와 병합 규칙(레거시 호환용)
● model.safetensors.index.json + model-00001-of-00004.safetensors ~ model-00004-of-00004.safetensors — 약 15GB(BF16) 가중치를 4개 shard로 분할 저장(safetensors 포맷 — 메모리 매핑 및 안전 로딩 보장)
● LICENSE — Apache-2.0 라이선스 본문
● README.md — 모델 카드(스펙·사용 예시·벤치마크·인용 정보)

06활용 가능 업무

경기도청 및 산하 공공기관 업무에 적용 가능한 구체 시나리오는 다음과 같다.

① 민원 1차 응대 챗봇 — FAQ·자주 묻는 질문을 RAG로 검색해 차량 등록·세금 납부·지원금 신청 절차를 자동 응답
② 내부 문서 요약 — 회의록·공문·보고서를 자동 요약하고 핵심 결정사항·후속 조치를 추출
③ 법령·조례 검색 및 해석 — 벡터 DB와 결합한 RAG 시스템에서 관련 법령을 인용한 뒤 평이한 해설 생성
④ 보고서 초안 자동 생성 — 통계 데이터·이슈 메모 입력으로 부서장 보고용 1차 초안 작성
⑤ 다국어 민원 통번역 — 외국인 주민 대상 한·영·중·일 민원 통번역 보조
⑥ 코드·SQL 작성 보조 — 데이터 분석 부서의 SQL·Python 스크립트 작성·디버깅 보조
⑦ 이메일·공문서 검수 — 맞춤법·문체·격식 점검 및 표준 양식 변환

배포 형태: vLLM·TGI·llama.cpp(GGUF)·Ollama 등 다양한 추론 엔진을 모두 지원하며, on-premise 배포로 민감 정보의 외부 유출을 최소화할 수 있다.

07한계와 주의사항

① 한국어 도메인 성능 한계 — 한국어 데이터 비중이 영어·중국어 대비 낮아, 법률·의료·공공행정 전문 용어에서 영어 대비 정확도 저하 가능. ② 사실성·환각(Hallucination) — 7B 규모 한계로 사실 환각 위험이 존재하며, RAG·인용 표시·근거 제시 같은 외부 보강이 필수. ③ 평가 데이터 한계 — 공식 벤치마크는 영어·중국어 중심이어서 KLUE·KoBEST·KMMLU 등 한국어 벤치마크와 공공 도메인 자체 평가셋으로 별도 검증 필요. ④ 학습 데이터 비공개 — 정확한 학습 데이터 구성·소스가 공개되지 않아 개인정보·저작권 보호 콘텐츠 포함 여부와 데이터 오염(test set leakage) 가능성을 완전히 배제하기 어려움. ⑤ 라이선스 제약 — Apache-2.0이지만 사용 약관(Acceptable Use Policy)에 군사·불법 행위 금지 조항이 포함되어 도입 전 약관 확인 필요. ⑥ 장컨텍스트 비용 — 32K → 128K 확장은 YaRN 설정이 필요하며 KV 캐시 메모리·추론 지연이 크게 증가. ⑦ 안전성·편향 — 다국어 학습 모델 특성상 특정 정치·사회 주제에서 편향이 나타날 수 있어 출력 필터링·redaction 모듈을 함께 운영 권장.

08종합 의견

경기도 및 산하 공공기관 업무 환경 기준으로 평가했을 때, Qwen2.5-7B-Instruct는 ① Apache-2.0 라이선스로 도입 시 법적 부담이 가장 낮고, ② 7.6B 규모로 단일 A100·H100 GPU 또는 4bit 양자화 시 RTX 4090 한 장으로 운용 가능해 도입 비용이 합리적이며, ③ 한국어·영어·다국어 동시 지원으로 외국인 민원·국제 협력 업무에도 활용 가능한 균형 잡힌 모델이다.

다만 실제 도입 전 다음 사항의 검증이 반드시 필요하다고 판단한다.
● 한국어 공공 벤치마크 자체 평가 — KLUE·KoBEST·KMMLU 기반 정량 평가
● 부서별 PoC — 부서별 실제 민원·문서 200~500건 샘플로 RAG 파이프라인 PoC
● 개인정보 보호 모듈 — PII 탐지·마스킹·출력 필터링 모듈 결합 검증
● 경쟁 모델 비교 — Llama-3.1-8B-Instruct, Gemma-2-9B-it, LG EXAONE-3.5-7.8B-Instruct 등 동급 모델과 한국어 정성·정량 비교

특히 한국어·한국 문화 특화 모델인 EXAONE 계열과의 비교 평가를 거쳐 도메인별(민원응대 vs 문서요약 vs 코드보조) 최적 모델을 분리 선정하는 것이 가장 합리적이라고 판단된다.
