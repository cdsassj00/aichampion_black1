01모델 선택 이유

현재 보유 중인 개인용 컴퓨팅 환경(로컬 환경)에서 원활하게 구동 가능한 최적의 파라미터 규모를 가졌으며, Google의 Gemma 아키텍처를 기반으로 하여 한국어 및 영어 지시행행 성능이 검증된 모델이기 때문입니다. 특히 4B(40억) 규모의 파라미터는 성능과 추론 속도 사이의 균형이 뛰어나 실제 서비스 프로토타입 제작에 적합하다고 판단했습니다.

03학습 데이터와 튜닝 방식

Google의 대규모 다국어 데이터셋을 기반으로 사전 학습(Pre-training)되었으며, 사람이 작성한 지시사항을 따르도록 SFT(Supervised Fine-Tuning)와 RLHF(Reinforcement Learning from Human Feedback)가 적용되었습니다. 특히 'E' 시리즈는 효율성(Efficiency)을 극대화하기 위해 최적화된 학습 레시피를 사용한 것으로 알려져 있습니다.

04파라미터 및 구조적 특징

ArchitectureGemma (Transformer-based, Decoder-only)
Context Length8192 Tokens (기본 설정 기준)
Hidden Size3072
Layers20 ~ 24 (모델 버전에 따라 상이)
Attention Heads8 (Grouped Query Attention 적용 가능성 포함)
TokenizerSentencePiece (Vocab Size: 256,000)

05모델 파일 구성

  • config.json: 모델의 레이어 수, 헤드 수, 활성화 함수 등 하이퍼파라미터 및 구조 정보가 포함된 설계도 역할.

  • tokenizer.json / .model: 텍스트를 모델이 이해할 수 있는 토큰(숫자)으로 변환하는 규칙 정의.

  • model.safetensors: 모델의 실제 가중치(Weights) 파일로, 보안성이 강화된 텐서 저장 형식.

  • generation_config.json: 텍스트 생성 시 온도(Temperature), Top-P 등 샘플링 전략 설정 포함.

  • special_tokens_map.json: <bos>, <eos>, <pad> 등 특수 토큰의 매핑 정보 포함.

    • 06활용 가능 업무

    기관 내 매뉴얼, 법령 등 로컬 환경에서 기존 문서 기반 질의응답(RAG) 수

    07한계와 주의사항

  • 한국어 맥락 유지: 7B 이상의 대형 모델에 비해 긴 한국어 문맥을 유지하는 능력이 다소 부족할 수 있음.

  • 환각(Hallucination): 지식 기반의 복잡한 질문에 대해 사실과 다른 내용을 그럴듯하게 답변할 위험 존재.

  • 추론 리소스: 4B 모델이지만 원활한 추론을 위해 최소 8GB~12GB 이상의 VRAM 권장.

    • 08종합 의견

    본 모델은 대형 언어 모델(LLM)을 도입하기 부담스러운 중소규모 조직이나 개인 개발자에게 최적의 선택지입니다. 특히 데이터 보안이 중요한 공공기관 내부망에서 가벼운 문서 요약 및 분류 업무에 바로 투입하기 적합합니다. 실제 도입 전에는 특정 도메인 데이터(행정 용어 등)를 활용한 추가적인 LoRA 미세 조정(Fine-tuning)을 통해 정확도를 검증하는 PoC 과정이 필요할 것으로 보입니다.
