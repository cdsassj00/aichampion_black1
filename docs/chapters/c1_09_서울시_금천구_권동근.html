01모델 선택 이유

1B~7B 범위 SLM 후보군을 라이선스·한국어 성능·보안성·시스템 호환성 네 가지 기준으로 평가한 결과, Kanana 1.5 2.1B Instruct가 모든 항목에서 합격선 이상의 균형을 충족하는 후보로 선정되었다.

검토 항목 평가 내용 결과
라이선스 Apache 2.0 — 내부망 활용·수정·파인튜닝·재배포 시 별도 계약 불필요 적합
한국어 성능 카카오 자체 한국어 코퍼스 기반. KOBEST·HAE-RAE 동급 파라미터 상위권. 행안부 AI 국민비서 시범서비스 공공 레퍼런스 존재 적합
보안성 가중치 1회 다운로드 후 완전 오프라인 구동. 주요 추론 엔진(vLLM·llama.cpp) 텔레메트리 비활성화 지원 적합
시스템 호환성 4-bit 양자화 시 VRAM 약 1.5 GB. 보급형 GPU(RTX 4060·NVIDIA L4) 운용 가능. 함수 호출(Function Calling) 및 RAG 파이프라인 연동 지원 적합
비교 탈락 후보 EXAONE 4.0 1.2B — 한국어 성능 우수하나 NC 라이선스(출력물 권리 귀속 조항)로 공문서 자동 생성 업무와 충돌 소지 제외

03학습 데이터와 튜닝 방식

사전학습 데이터 (Pre-training Data)
  • 총 3조(3T) 토큰. 출처는 모두 공개 데이터이며 카카오 자체 서비스 데이터는 미포함.
  • 구성 카테고리: English Web, Korean Web, Academic, Code, Encyclopedia, Instruction Data
  • Stage 1 (2.7T): English Web 67.8% 중심 / Stage 2 (300B): Academic·Code 중심, Korean Web·Instruction 포함
  • 데이터 품질 관리: deduplication → heuristic filtering → PII anonymization → 언어별 모델 기반 필터링(영어: DCLM classifier, 한국어: 자체 edu filter/FastText 기반)
포스트학습 데이터 (Post-training Data)
  • SFT: 총 120만 인스턴스 (Code 41.1%, Math 21.7%, Instruction Following 19.2%, General Chat 10.0%, Safety 8.0%)
  • Preference: 총 40.7만 인스턴스 (Instruction Following 32.9%, Code 31.7%, General Chat 23.3%, Math 12.0%)
  • 데이터 출처: 공개 데이터셋 + 인간 기여자(human contributors) 혼합
튜닝 방식
  • Pre-training: Staged Pre-training (2단계), Depth Up-scaling (DUS), Pruning & Distillation
  • Post-training: SFT (Supervised Fine-Tuning)
  • Preference Optimization: Offline DPO → Online DPO (비동기 응답 샘플링, 고정 reference 모델 유지)
  • Reward Model: Bradley-Terry 모델 기반, Best-of-N 정책으로 선정
  • 특화 적용: Embedding: LLM2Vec / RAG: SFT+DPO+모델 병합 / Function Calling: 도메인 사전학습+SFT

05모델 파일 구성

  • LICENSE: 모델의 라이선스 정보(Apache 2.0)를 담고 있으며, 사용 권한 및 제약 사항을 명시하는 파일입니다.
  • README.md: 모델에 대한 전반적인 설명, 아키텍처, 성능 벤치마크 지표, 사용 방법 등을 설명하는 안내 문서 파일입니다.
  • config.json: 모델의 아키텍처 구조와 하이퍼파라미터(예: 레이어 수, 히든 사이즈 등)를 정의하는 핵심 설정 파일입니다.
  • generation_config.json: 모델이 텍스트를 생성할 때 기본적으로 사용되는 설정(예: 최대 토큰 수, Temperature 등)을 담고 있는 파일입니다.
  • gitattributes: Git 대용량 파일 저장소(LFS) 처리 방식 등 Git 저장소의 파일 속성을 지정하는 파일입니다.
  • special_tokens_map.json: 문장의 시작, 끝, 패딩 등을 나타내는 특수 토큰(Special Tokens)들의 매핑 정보를 담고 있는 파일입니다.
  • tokenizer.json: 모델이 텍스트를 이해할 수 있도록 단어를 쪼개는 토크나이저의 전체 어휘 사전(Vocabulary)과 병합 규칙을 담고 있는 파일입니다.
  • tokenizer_config.json: 토크나이저를 초기화하고 구동하는 데 필요한 상세 설정값들을 담고 있는 파일입니다.
  • 모델.safetensors: 모델의 실제 신경망 가중치가 담긴 핵심 파일

06활용 가능 업무

운용 원칙

본 모델은 실무자 검토 전 '1차 자동 처리' 역할에 한정하며, 법적 효력을 갖는 최종 판단은 실무자가 수행한다.

① 행정 문서·보고서·회의록 자동 요약
  • 기본 32K 토큰(YaRN 적용 시 최대 128K) 입력 지원 — 1~2시간 회의록 단일 처리 가능
  • 안건·결정 사항·후속 조치를 구조화 형태로 출력
  • 회의 오디오 처리 시 별도 STT 도구(Whisper 등)와 파이프라인 연계 필요
② 문서 카테고리 자동 분류
  • 접수 민원 부서·담당자 자동 배정, 공문 유형 분류(시행·보고·통보·요청)
  • 소형 모델이 가장 안정적으로 수행하는 영역 — LoRA 파인튜닝 적용 시 정확도 추가 향상
③ 공문서 양식 초안 자동 생성
  • 함수 호출(Function Calling) 기반 JSON 구조 출력 → 결재문서·시행문·보도자료 양식 자동 채우기
  • 회의 요약 데이터 입력 → '제목-수신-발신-주요 내용-붙임' 형식 초안 생성
  • 출력물에 'AI 1차 작성 / 실무자 검토 필요' 메타데이터 부착 의무화
공통 파이프라인

세 가지 업무는 동일한 RAG 파이프라인으로 통합 운용 가능. 자치법규·행정해석·법령 고시를 벡터 DB에 구축하여 생성 근거를 실시간 보강.

07한계와 주의사항

성능 한계 — 환각(Hallucination) 위험
  • 법령 조항 번호 오인용·수치 임의 추정·단정적 법률 진술 발생 가능
  • 대응 : RAG로 외부 근거 강제 제공 + 근거 없는 답변 차단 시스템 프롬프트 적용 + 실무자 사후 검증 의무화
  • 허가 여부 결정·행정처분 사유 작성 등 법적 판단 업무는 적용 범위에서 제외
보안 운영
  • 외부망 다운로드 → 악성코드 스캔·SHA256 해시 검증 → 행정망 반입 절차 필수
  • 추론 로그 개인정보 마스킹 처리 후 보존 정책 수립
  • 민원인 제출 텍스트 처리 시 프롬프트 인젝션 방지 입력 정제 로직 선행 적용
생태계 변화 대응
  • SLM은 6~12개월 단위로 동급 이상 모델 출시 중 — 모델 교체를 전제한 분리형 설계 필수
  • 모델 가중치·RAG 벡터 DB·LoRA 어댑터·비즈니스 로직 계층을 분리하여 모델만 교체해도 축적 자산 재활용 가능하도록 구성
  • 법령 개정 시 수시, 주요 고시 변경 시 즉시 RAG 벡터 DB 갱신 일정 운영 매뉴얼 명시

08종합 의견

본 모델은 7B 이하 후보군 중 라이선스 안전성·한국어 성능·오프라인 구동·공공 레퍼런스·자원 효율성을 동시에 충족하는 가장 균형 있는 선택지로 판단된다. 단, 모델 단독 도입만으로 행정 업무 즉시 투입 수준의 정확도는 확보되지 않으며, 아래 체계가 병행 구축되어야 실효성이 확보된다.
