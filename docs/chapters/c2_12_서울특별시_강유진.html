01모델 선택 이유

본 모델은 3B(30억)라는 경량화된 파라미터 사이즈에도 불구하고, 이전 세대의 7B 모델을 상회하는 한국어 처리 능력과 논리 추론 성능을 보유하고 있습니다. 특히 공공기관의 민원 응대나 내부 행정 문서 요약과 같이 보안이 중요하여 온프레미스(On-premise) 환경 구축이 필요한 업무에서 저사양 하드웨어로도 고성능 AI 서비스를 구현할 수 있어 최적의 선택지라 판단했습니다.

02모델 기본 정보

모델명Qwen2.5-3B-Instruct
개발 주체Alibaba Cloud (Qwen Team)
모델 규모3.09B (약 31억 파라미터)
라이선스Apache-2.0
모델 유형Causal Language Model (Instruct-tuned)

03학습 데이터와 튜닝 방식

  • 사전학습: 18조 개의 토큰(18T tokens)으로 구성된 대규모 데이터셋으로 학습되었습니다. 여기에는 방대한 양의 다국어(한국어 포함), 코드, 수학 데이터가 포함되어 있습니다.

  • 튜닝 방식: * SFT (Supervised Fine-tuning): 지시어 이행 능력을 극대화하기 위한 지도 학습 적용.

    • RLHF (Reinforcement Learning from Human Feedback): 인간의 선호도에 부합하도록 정렬하는 강화학습 적용.

    • DPO (Direct Preference Optimization): 모델의 답변 품질을 높이기 위한 최신 최적화 기법 도입.

    • 04파라미터 및 구조적 특징

    ArchitectureRoPE, SwiGLU, RMSNorm 및 Attention QKV bias가 적용된 Transformer 구조
    Context Length128,000 토큰 (실제 생성은 8,192 토큰 최적화)
    Hidden Size2048
    Layers36
    Attention Heads16 (Query) / 2 (Key-Value, GQA 적용)
    TokenizerBPE (Byte Pair Encoding) 기반, Vocab size: 151,643

    05모델 파일 구성

  • config.json: 모델의 아키텍처 정보(레이어 수, 히든 사이즈 등)가 담긴 핵심 설정 파일.

  • tokenizer.json / .model: 텍스트를 AI가 이해하는 숫자(토큰)로 변환하는 규칙 정의.

  • model.safetensors: 모델의 실제 가중치(Weights) 데이터. 보안성이 높은 최신 포맷 사용.

  • generation_config.json: 답변 생성 시의 온도(Temperature), Top-p 등 기본 샘플링 전략 설정.

  • special_tokens_map.json: 문장의 시작, 끝, 시스템 프롬프트 등을 구분하는 특수 토큰 매핑 정보.

    • 06활용 가능 업무

    본 모델은 서울시 데이터전략과의 '데이터 기반 행정 혁신'을 위한 AI 빅데이터 분석 기획 자동화서비스 핵심 엔진으로 활용 가능합니다. 구체적으로 다음과 같은 업무 자동화 프로세스를 구축할 수 있습니다.

    • 사업계획서 자동 분석 및 구조화: 실국 공무원이 업로드한 한글(.hwpx) 기반 사업계획서에서 핵심 요구사항, 분석 대상, 기대 효과 등을 추출하여 표준화된 데이터 구조로 변환합니다.

    • 지능형 데이터 매칭 (RAG 연합): 서울시가 보유한 KT 생활인구, 신한카드 소비 데이터, KCB 유동인구 및 공공데이터 정의서(Data Dictionary)를 벡터 DB화하여, 사업 목적에 가장 적합한 데이터셋을 추천하고 분석 변수를 정의합니다.

    • 분석기획서 초안 생성: 과거의 우수 분석 사례(AI-Readable 기획서)를 학습/참조하여, 신규 사업에 대한 '분석 목적 - 데이터 설계 - 분석 방법론 - 시각화 방안'이 포함된 표준 기획서 초안을 생성합니다.

    07한계와 주의사항

  • 데이터 최신성 및 환각: 모델이 학습하지 못한 최신 공공데이터 수치나 법령에 대해 잘못된 정보를 제공할 수 있습니다. 이를 방지하기 위해 반드시 내부 벡터 DB를 통한 RAG(검색 증강 생성) 기법을 병행하여 근거 기반의 답변을 생성해야 합니다.

  • 보안 및 개인정보: 분석 기획 과정에서 비식별화되지 않은 민감 정보가 입력되지 않도록 전처리 필터링 가이드라인이 필요합니다.

  • 도메인 특화 용어: 일반적인 모델은 서울시만의 특수한 행정 용어나 데이터 약어를 오해할 수 있으므로, 제공해주신 데이터 정의서 기반의 정밀한 프롬프트 엔지니어링이 필수적입니다.

    • 08종합 의견

    Qwen2.5-3B-Instruct 모델은 서울시 내부 행망 플랫폼에 탑재하기에 가장 전략적인 선택입니다.

    1. 자원 효율성: 7B 이상의 모델 대비 낮은 사양의 GPU에서도 빠른 추론이 가능하여, 다수의 공무원이 동시 접속하는 서비스 플랫폼 구축 시 비용 효율적입니다.

    2. RAG 최적화: 128K에 달하는 넓은 컨텍스트 창을 지원하여, 방대한 데이터 정의서와 과거 기획서 전문을 참조(Context)로 입력하더라도 누락 없는 분석이 가능합니다.

    3. 전문성 강화: 데이터전략과가 보유한 AI Readable한 고품질의 공문서(.hwpx) 데이터와 민간(카드/통신/신용/융합), 공공(교통/안전/관광) 데이터를 결합한 '서울형 분석 컨설팅 에이전트'로 발전시킬 경우, 공무원들의 분석 기획 문턱을 획기적으로 낮추고 데이터 기반 의사결정 문화를 확산시키는 핵심 동력이 될 것입니다.
