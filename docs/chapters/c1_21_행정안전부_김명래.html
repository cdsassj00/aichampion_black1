01모델 선택 이유

EXAONE-3.5-2.4B-Instruct를 선택한 가장 큰 이유는 한국어 특화 성능과 공공기관 환경에의 적합성입니다. 행정안전부 업무는 한국어 행정 문서 처리, 민원 응대, 내부 보고서 작성 등 한국어 중심의 작업이 대부분을 차지합니다. EXAONE 3.5는 LG AI Research에서 개발한 영한 이중언어 모델로, 동급 규모 모델 중 한국어 벤치마크(KoMT-Bench: 7.24, LogicKor: 8.51)에서 Qwen 2.5 3B, Llama 3.2 3B, Gemma 2 2B 등 경쟁 모델을 크게 앞서는 성능을 보입니다. 또한 2.4B 규모의 경량 모델로 GPU 자원이 제한된 온프레미스 환경에서도 원활한 배포가 가능하며, 32K 토큰의 긴 컨텍스트 처리를 지원해 장문의 행정 문서 분석에도 적합합니다. 공공기관의 데이터 보안 요건상 외부 클라우드 API 의존도를 낮추고 내부 서버에서 직접 운용해야 하는 상황에서, 경량이면서도 한국어 성능이 우수한 EXAONE-3.5-2.4B-Instruct는 행정안전부 업무 환경에 최적화된 선택입니다.

03학습 데이터와 튜닝 방식

EXAONE 3.5의 사전학습 데이터 구성은 공개된 기술 보고서(arXiv: 2412.04862) 기준으로 영어와 한국어 이중언어 코퍼스를 중심으로 구성되었습니다. 다양한 공개 웹 데이터, 학술 자료, 코드, 한국어 특화 데이터셋 등이 포함된 것으로 알려져 있으나, 세부 데이터 구성 비율은 공개 범위 내에서는 확인 어려움입니다. 튜닝 방식으로는 Instruction Tuning(SFT, Supervised Fine-Tuning)을 기반으로 하며, 실제 사용 시나리오에 맞는 지시 따르기(instruction following) 능력을 강화하는 방향으로 학습되었습니다. 또한 모델 카드에서 시스템 프롬프트 활용을 강력히 권장하고 있어, RLHF 또는 DPO 계열의 선호도 정렬(alignment) 기법도 적용된 것으로 추정됩니다. 모델은 Transformers 라이브러리 v4.43 이상 환경에서 bfloat16 정밀도로 구동하도록 최적화되어 있습니다.

05모델 파일 구성

Hugging Face 저장소(LGAI-EXAONE/EXAONE-3.5-2.4B-Instruct)의 주요 파일 구성은 다음과 같습니다. config.json은 모델 아키텍처 전체 설정(레이어 수, hidden size, attention heads 등)을 정의하는 핵심 파일입니다. tokenizer.json 및 tokenizer_config.json은 텍스트를 토큰으로 변환하는 규칙과 특수 토큰 설정을 담고 있으며, vocab_size 102,400의 어휘를 관리합니다. model.safetensors(또는 model-00001-of-XXXXX.safetensors 형태의 분할 파일)는 실제 학습된 가중치를 안전하게 저장한 파일로, 기존 .bin 파일 대비 보안성과 로딩 속도가 개선된 포맷입니다. generation_config.json은 텍스트 생성 시 사용하는 기본 파라미터(max_new_tokens, temperature, top_p 등)를 설정합니다. special_tokens_map.json은 [BOS], [EOS], [PAD] 등 모델이 사용하는 특수 토큰의 매핑 정보를 담고 있습니다. configuration_exaone.py는 EXAONE 전용 아키텍처 코드로, trust_remote_code=True 옵션으로 로드할 때 사용됩니다.

06활용 가능 업무

행정안전부 업무 환경에서 EXAONE-3.5-2.4B-Instruct를 다음과 같이 활용할 수 있습니다. 첫째, 민원 응대 챗봇으로 활용 가능합니다. 국민이 온라인 민원 시스템(정부24 등)에서 질문을 입력하면 모델이 관련 법령·절차를 한국어로 안내하는 자동 응답 시스템을 구축할 수 있습니다. 둘째, 내부 행정 문서 요약 및 분석에 적합합니다. 32K 토큰의 긴 컨텍스트를 활용해 보고서, 회의록, 법령 문서 등 장문 행정 문서를 핵심 내용 중심으로 요약하거나 질의응답 형태로 처리할 수 있습니다. 셋째, RAG(Retrieval-Augmented Generation) 기반 내부 지식베이스 검색 시스템과 연동해 부서별 업무 매뉴얼, 규정집 등을 실시간 검색·답변하는 내부 AI 도우미로 활용할 수 있습니다. 넷째, 보고서 초안 작성 지원으로 담당자가 키워드나 개요를 입력하면 행정 보고서 초안을 자동 생성해 업무 효율을 높일 수 있습니다. 다섯째, vLLM, llama.cpp, Ollama 등 다양한 추론 프레임워크를 지원해 기존 온프레미스 서버 환경에 유연하게 통합 배포가 가능합니다.

07한계와 주의사항

EXAONE-3.5-2.4B-Instruct 사용 시 다음과 같은 한계와 주의사항을 고려해야 합니다. 첫째, 환각(Hallucination) 가능성입니다. 모델 카드에서 명시하듯 학습 데이터 통계에 의존하는 특성상 사실과 다른 내용을 그럴듯하게 생성할 수 있어, 법령 해석이나 정책 정보 제공 시 반드시 인간 검수가 필요합니다. 둘째, 라이선스 제한입니다. EXAONE AI Model License 1.1 - NC 조건에 따라 상업적 목적으로의 직접 활용에는 제한이 있을 수 있으므로, 상업적 서비스 전환 시 LG AI Research와 별도 협의가 필요합니다. 셋째, 2.4B 규모 경량 모델의 특성상 복잡한 추론, 전문 법률 해석, 수치 계산 등에서는 7B 이상 모델 대비 정확도가 떨어질 수 있습니다. 넷째, 학습 데이터 편향 문제로 연령, 성별, 인종 등과 관련한 편향된 응답이 생성될 가능성이 있어 민감한 민원 응대에 주의가 필요합니다. 다섯째, 모델이 최신 정보를 반영하지 못하므로 최신 법령 개정 사항이나 시사 정보에 대해서는 RAG 등 외부 검색 연동이 필수입니다.

08종합 의견

EXAONE-3.5-2.4B-Instruct는 행정안전부 업무 환경에 단계적으로 도입하기에 적합한 모델이라고 판단합니다. 특히 데이터 보안이 중요한 공공기관 특성상 외부 클라우드 API 없이 내부 서버에서 자체 운용할 수 있는 온프레미스 배포 모델로서의 가치가 높습니다. 도입 적합 조건으로는 GPU 메모리 8GB 이상의 내부 서버 환경, 민원 응대 또는 내부 문서 요약 등 명확히 정의된 단일 과제 중심의 초기 적용, 그리고 인간 검수 프로세스가 병행되는 환경을 권장합니다. 추가 검증 및 POC 방향으로는 행정안전부 실제 민원 데이터를 활용한 질의응답 정확도 평가, 기존 담당자 대비 문서 요약 품질 비교 테스트, 법령 기반 RAG 파이프라인 연동 실험이 필요합니다. 장기적으로는 행정 도메인 특화 데이터로 LoRA 파인튜닝을 적용하면 공공행정 특화 성능을 더욱 끌어올릴 수 있을 것으로 기대됩니다. 다만 상업적 활용 시 라이선스 조건을 반드시 재검토해야 하며, 초기에는 내부 업무 지원 보조 도구로 한정해 리스크를 최소화하는 점진적 도입을 권장합니다.
