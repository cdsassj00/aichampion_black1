01모델 선택 이유

EleutherAI의 Polyglot-Ko-5.8B는 1B~7B 범위에 포함되는 한국어 중심 공개 언어모델입니다. Apache-2.0 라이선스로 공개되어 연구·실험·업무 적용 검토가 비교적 용이하고, 한국어 말뭉치 기반의 autoregressive language model이므로 행정 문서 초안 작성, 민원 질의응답, 내부 지식검색 RAG의 생성 모델 후보로 분석할 가치가 큽니다. 또한 5.8B 규모는 단일 고성능 GPU 또는 양자화 환경에서 PoC를 수행하기에 현실적인 크기입니다.

03학습 데이터와 튜닝 방식

모델 카드는 Polyglot-Ko-5.8B가 한국어 중심의 autoregressive language model이며 GPT-NeoX 기반으로 학습되었다고 설명합니다. 공개 정보 기준으로 약 1,720억 개 한국어 토큰을 사용해 320,000 step 동안 학습되었고, 학습에는 256개의 A100 GPU가 사용된 것으로 안내되어 있습니다. 이 모델은 기본형 Base 모델이므로 Instruction Tuning, SFT, RLHF, DPO, LoRA, QLoRA가 기본 모델 자체에 적용되었다고 보기 어렵습니다. 따라서 챗봇처럼 지시를 따르는 응답을 안정적으로 생성하려면 별도의 SFT 또는 instruction fine-tuning이 필요합니다. 데이터의 세부 원천, 필터링 기준, 공공·행정 도메인 데이터 포함 여부는 공개 범위 내에서 제한적으로만 확인됩니다.

05모델 파일 구성

Hugging Face 저장소의 config.json은 모델 구조를 정의하는 핵심 설정 파일입니다. 여기에는 hidden_size 4096, num_hidden_layers 28, num_attention_heads 16, max_position_embeddings 또는 n_ctx 2048과 같은 구조적 값이 포함됩니다. tokenizer 관련 파일은 한국어 텍스트를 토큰 ID로 변환하고 다시 문자열로 복원하는 역할을 합니다. model.safetensors 또는 shard 파일은 실제 학습된 가중치를 담고 있으며, 안전한 텐서 저장 형식으로 로드됩니다. generation_config.json이 있는 경우에는 max_new_tokens, temperature, top_p, repetition_penalty 같은 생성 기본값을 지정할 수 있습니다. special_tokens_map.json은 BOS, EOS, padding 등 특수 토큰의 의미와 매핑을 관리합니다.

06활용 가능 업무

공공기관 업무에서는 Polyglot-Ko-5.8B를 단독 답변 생성기보다는 RAG 구조의 한국어 생성 모델 후보로 활용하는 것이 적합합니다. 예를 들어 행정 지침, 조례, 업무 매뉴얼, FAQ를 벡터 검색으로 찾은 뒤 검색 근거를 바탕으로 민원 답변 초안을 작성할 수 있습니다. 내부 문서 요약, 회의록 정리, 보고서 초안 작성, 정책 문서의 질의응답, 반복 민원 응대 시나리오 생성에도 활용 가능성이 있습니다. 다만 Base 모델이므로 실제 서비스에서는 instruction tuning, 안전성 필터, 개인정보 마스킹, 근거 문장 인용, 답변 검증 절차를 함께 설계해야 합니다.

07한계와 주의사항

Base 모델이기 때문에 사용자의 지시를 항상 안정적으로 따르지 못할 수 있고, 사실과 다른 내용을 생성하는 환각 가능성이 있습니다. 컨텍스트 길이가 2,048 토큰 수준으로 비교적 짧아 긴 행정 문서 전체를 한 번에 처리하기 어렵습니다. 학습 데이터의 세부 구성과 필터링 방식이 완전히 공개되어 있지 않으므로 특정 도메인 편향이나 최신성 한계가 존재할 수 있습니다. Apache-2.0 라이선스는 활용성이 높지만, 실제 업무 적용 시에는 입력 데이터의 개인정보·보안 등급·저작권을 별도로 검토해야 합니다. 또한 최신 instruction 모델과 비교하면 대화 품질, 도구 사용, 장문 추론 능력은 낮을 수 있습니다.

08종합 의견

Polyglot-Ko-5.8B는 한국어 중심 공개 SLM을 이해하고 파인튜닝 실험을 수행하기에 적합한 기준 모델입니다. 공공 업무에 바로 투입하기보다는 기관 내부 문서로 RAG를 구성하고, 민원·행정 문체에 맞춘 SFT를 진행한 뒤 정확성·안전성·개인정보 비식별화 평가를 거치는 방식이 적절합니다. 특히 한국어 데이터 기반이라는 장점과 Apache-2.0 라이선스의 개방성은 장점이지만, Base 모델이라는 특성상 지시 수행 능력은 추가 튜닝으로 보완해야 합니다. 따라서 본 모델은 ‘한국어 공공업무용 LLM PoC의 베이스라인’으로 가장 적합하다고 판단합니다.
