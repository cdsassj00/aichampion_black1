01 모델 선택 이유

이번 과제에서는 Qwen/Qwen2.5-3B-Instruct를 선택하였다. 이 모델은 3B 규모로 과제 조건인 1B~7B 범위에 정확히 부합하며, 단순한 Base 모델이 아니라 지시 수행용(Instruct)으로 후처리된 실무형 모델이라는 점이 장점이다. 또한 긴 문맥 처리, 표·구조화 데이터 이해, JSON 형식 출력, 다국어 지원 등 실제 행정 업무에서 바로 연결해 볼 수 있는 특성이 분명하다. 공공기관 현장에서는 초거대 모델보다도 적당한 크기와 활용 가능성의 균형이 중요하므로, 학습용·업무 실험용 분석 대상으로 적합하다고 판단하였다. 다만 본 모델은 Qwen Research License가 적용되므로 즉시 운영 서비스에 투입하는 상용 도입 모델이라기보다, SLM 구조와 활용 가능성을 분석하는 학습·연구용 과제 대상으로 선택했다는 점을 함께 밝힌다.

03 학습 데이터와 튜닝 방식

Qwen2.5 시리즈는 공식 블로그와 기술 리포트 기준으로 최신 대규모 데이터셋(기존 7조 토큰에서 확장된 최대 18조 토큰 규모)을 활용해 사전학습되었다. 다만 개별 데이터셋 이름과 비율이 모두 상세 공개된 것은 아니므로, 실제 보고서에서는 “대규모 사전학습 데이터 기반”과 “세부 구성은 제한적으로 공개”라는 점을 함께 적는 것이 정확하다.

  • 사전학습(Pretraining): 대규모 텍스트 코퍼스를 기반으로 언어 이해·생성 능력 형성
  • 사후학습(Post-training): 지시 수행 성능 향상을 위한 Instruct 계열 후처리 적용
  • 튜닝 성격: 사용자 질문을 잘 따르고, 긴 답변·표·JSON 같은 구조화 응답에 강하도록 개선
  • 언어 범위: Qwen2.5 시리즈는 29개 이상의 언어를 지원하며 한국어도 포함됨
  • 해석: 행정 문서 요약, 질의응답, 정리형 보고서 작성처럼 “명확한 지시를 주고 결과를 받는” 업무에 적합한 구조

모델 카드만 보면 세부 튜닝 단계가 축약돼 있지만, Qwen2.5 Technical Report(arXiv:2412.15115)에서는 사후학습 단계에 대해 100만 개 이상의 SFT(supervised fine-tuning) 샘플과 다단계 강화학습이 적용되었다고 설명한다. 구체적으로는 offline learning DPO와 online learning GRPO가 포함된 multistage reinforcement learning이 명시되어 있어, 본 모델은 단순한 SFT 모델이 아니라 SFT + DPO + 온라인 RL 기반으로 정렬된 Instruct 모델로 이해할 수 있다.

04 파라미터 및 구조적 특징

ArchitectureQwen2ForCausalLM (decoder-only Transformer)
세부 구조RoPE, SwiGLU, RMSNorm, Attention QKV bias, tied word embeddings
Context Length32,768 tokens (모델 카드상 생성은 최대 8,192 tokens)
Hidden Size2048
Intermediate Size11008
Layers36
Attention Heads16 heads / KV heads 2 (GQA 구조)
TokenizerQwen 전용 tokenizer 사용, vocab size 151,936
기술적 해석중형 이하 크기이지만 문맥 길이가 길고, 구조화 출력 및 멀티턴 대화에 유리한 설계가 반영되어 있다.

주요 평가 결과

지표공식 자료 해석
MMLUQwen2.5 시리즈는 공식 블로그 기준 85+ 수준의 지식 성능을 제시한다. 특히 3B 모델도 “3B급인데도 매우 경쟁력 있는 성능”의 대표 사례로 언급된다.
HumanEvalQwen2.5 계열은 코딩 역량 지표에서 85+ 수준의 향상을 강조한다. 이는 단순 대화형 응답뿐 아니라 코드 생성·자동화 보조에도 강점이 있음을 시사한다.
MATH수학 추론 관련 성능도 공식 블로그에서 80+ 수준으로 소개되며, 논리적 단계 정리·계산 보조 역량 향상을 보여준다.
Instruction Following공식 자료는 긴 글 생성, 표 해석, JSON 출력, 다양한 시스템 프롬프트 대응력이 크게 향상되었다고 설명한다. 이는 행정 업무에서 요구하는 구조화 응답과 지시 이행 능력과 직접 연결된다.

참고: 위 수치는 Qwen2.5 시리즈 전체 성능 설명과 기술 리포트의 대표 벤치마크 설명을 기반으로 정리하였다. 개별 3B 모델의 모든 세부 지표가 모델 카드에 표 형태로 완전 공개된 것은 아니므로, 보고서에서는 “시리즈 차원의 공식 성능 설명 + 3B 모델의 상대적 위치”를 함께 해석하였다.

05 모델 파일 구성

Hugging Face 저장소의 주요 파일은 아래와 같은 의미를 가진다.

README.md모델 카드. 모델 소개, 사용법, 특징, 주의사항이 들어 있다.
LICENSE라이선스 문서. 이 모델은 Qwen Research License를 따르므로 사용 범위를 반드시 확인해야 한다.
config.json모델 구조 정보가 담긴 핵심 설정 파일. hidden size, layer 수, attention head, context length 등을 확인할 수 있다.
generation_config.json텍스트 생성 기본값 설정 파일. temperature, top_p, top_k, repetition penalty 같은 생성 스타일이 담겨 있다.
tokenizer.json / tokenizer_config.json / merges.txt입력 문장을 토큰 단위로 쪼개는 규칙과 special token, 채팅 템플릿 등을 담고 있다.
model-00001-of-00002.safetensors
model-00002-of-00002.safetensors		실제 모델 가중치가 저장된 파일이다. 용량이 커서 분할 저장되어 있다.
model.safetensors.index.json분할된 가중치 파일의 인덱스 역할을 한다. 어떤 파라미터가 어느 파일에 들어 있는지 연결한다.

즉, 모델 카드는 “설명서”, config는 “설계도”, tokenizer는 “문장 해석 규칙”, safetensors는 “실제 모델 본체”라고 이해하면 쉽다.

사용 예시

모델 카드에 제시된 대표 사용 예시는 아래와 같다. 실무에서는 이 코드를 바탕으로 로컬 테스트나 서버 추론 환경을 구성할 수 있다.

from transformers import AutoModelForCausalLM, AutoTokenizer

model_name = "Qwen/Qwen2.5-3B-Instruct"
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype="auto",
    device_map="auto"
)
tokenizer = AutoTokenizer.from_pretrained(model_name)

messages = [
    {"role": "system", "content": "You are Qwen, created by Alibaba Cloud."},
    {"role": "user", "content": "학교 행정실 업무를 5줄로 요약해줘."}
]
text = tokenizer.apply_chat_template(messages, tokenize=False, add_generation_prompt=True)

06 활용 가능 업무

  • 행정 문서 요약: 공문, 회의자료, 계획서 핵심 정리
  • 민원 응대 초안 작성: 자주 묻는 질문에 대한 1차 답변 초안 작성
  • 규정·지침 질의응답: 내부 규정집, 업무 매뉴얼 기반 질의응답 시스템 구성
  • 회의록 정리: 회의 메모를 구조화하여 요약본 또는 실행 항목으로 정리
  • 보고서/보도자료 초안: 일정한 형식의 초안 생성 및 문장 다듬기
  • RAG 연계: 교육청 내부 자료를 검색해 답하는 문서 기반 챗봇의 언어모델로 활용 가능
  • 표·JSON 출력 기반 자동화: 일정 목록, 업무 체크리스트, 정형 데이터 결과물 생성에 유리

특히 이 모델은 구조화 출력 강점이 있어, 공공기관에서 필요한 정리·분류·요약형 업무에 잘 맞는다.

07 한계와 주의사항

  • 환각(Hallucination): 사실과 다른 내용을 그럴듯하게 생성할 수 있으므로 최종 검토가 필요하다.
  • 최신성 한계: 웹 검색이 자동으로 연결된 모델이 아니므로 최신 정책·법령은 별도 확인이 필요하다.
  • 공공업무 검증 필요: 대민 안내문, 공문 문안, 법적 해석은 반드시 담당자 검토가 선행되어야 한다.
  • 라이선스 제한: Qwen Research License 기반이므로 상업적·운영 서비스 적용 전 라이선스 검토가 필요하다.
  • 개인정보 주의: 민감한 행정자료를 그대로 입력하는 방식은 보안정책 검토 없이 사용하면 위험하다.
  • 한국 공공 도메인 특화 한계: 한국 행정 문맥에 특화된 전용 모델은 아니므로 추가 미세조정 또는 RAG 보완이 효과적일 수 있다.

08 종합 의견

Qwen2.5-3B-Instruct는 비교적 작은 3B 규모임에도 긴 문맥 처리, 구조화 출력, 다국어 지원, 지시 수행 능력을 고르게 갖춘 실용적 SLM이다. 초거대 모델만큼의 압도적 정확성을 기대하기는 어렵지만, 교육청·행정기관의 문서 요약, 질의응답, 초안 작성, 내부 자료 기반 챗봇 실험 같은 영역에서는 충분히 검토할 가치가 있다. 특히 과제 관점에서는 모델 카드와 설정 파일을 통해 기술적 구조를 이해하기 좋고, 실무 활용 시나리오를 연결하기도 쉬워 분석 대상으로 적합하다. 따라서 본 모델은 “공공업무 활용 가능성을 시험해 보기 좋은 중소형 Instruct 모델”로 평가할 수 있다.
