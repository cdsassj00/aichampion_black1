01모델 선택 이유

EXAONE-3.5-2.4B-Instruct를 선택한 가장 큰 이유는 국내 대기업(LG AI Research)이 개발한 공개 한국어 특화 SLM이라는 점입니다. 공공·행정 업무에서 AI를 실제로 활용하려면 한국어 처리 품질이 가장 중요한 조건인데, 동급 크기(2~3B) 모델 중 한국어 벤치마크(KoMT-Bench 7.24점, LogicKor 8.51점)에서 압도적 1위를 기록하고 있습니다.

두 번째 이유는 로컬 온프레미스 배포 가능성입니다. bfloat16 기준 약 4.8GB로, llama.cpp 또는 Ollama를 통해 인터넷 연결 없이 내부망에서 구동할 수 있습니다. 클라우드 외부 전송이 어려운 보안 환경에서도 AI를 활용할 수 있다는 점이 실무적으로 중요합니다.

세 번째로, 상세한 모델 카드·기술 보고서(arXiv 2412.04862)·config.json 등 설정 파일이 모두 공개되어 있어 이번 과제의 분석 목적에도 적합합니다. HuggingFace 모델 트리에는 이미 22개의 LoRA 어댑터와 26개의 파인튜닝 파생 모델이 등록되어 있어 커뮤니티 활용도도 높습니다.

03학습 데이터와 튜닝 방식

【사전학습 데이터】 구체적인 데이터셋 목록은 공개되지 않았습니다. 기술 보고서(arXiv 2412.04862)에 따르면 한국어·영어 혼합 대용량 코퍼스(웹 텍스트, 도서, 학술 자료, 코드 등)로 사전학습을 수행했습니다. 어휘 크기(vocab 102,400개)가 일반 모델(32K~50K)보다 약 2배인 것은 한국어 형태소 체계를 충분히 커버하기 위한 설계로 풀이됩니다. 산업·공공 특화 데이터 포함 여부는 공개 범위 내에서는 확인 어렵습니다.

【튜닝 방식】 Instruction Tuning(SFT): 사전학습 완료 후, 사람이 작성한 질문-응답 쌍(instruction–response pair)으로 Supervised Fine-Tuning을 수행했습니다. 모델 카드는 시스템 프롬프트("You are EXAONE model from LG AI Research, a helpful assistant.") 사용을 강력 권장하며, 이는 역할 인식·안전한 응답 생성이 학습되어 있음을 시사합니다.

【RLHF·DPO 적용 여부】 모델 카드에 명시되지 않습니다. 다만 Arena-Hard(48.2), AlpacaEval(37.1) 등 실사용자 선호도 벤치마크에서 동급 대비 강한 성능을 보여, 인간 피드백 기반 정렬(alignment)이 일정 수준 이상 적용된 것으로 추정됩니다.

【LoRA·QLoRA 파인튜닝 지원】 HuggingFace 모델 트리에 22개의 LoRA 어댑터가 등록되어 있어, 도메인 특화 데이터로 추가 파인튜닝이 검증된 상태입니다. 공식 컬렉션에서 AWQ(4비트 양자화)와 GGUF(llama.cpp 호환) 양자화 모델도 제공합니다.

04파라미터 및 구조적 특징

ArchitectureExaoneForCausalLM — LG AI Research 정의 커스텀 Decoder-only Transformer. 로드 시 trust_remote_code=True 필요 (configuration_exaone.py 자동 실행)
Context Length32,768 tokens — RoPE Llama3 방식 스케일링(factor=8.0)으로 원래 8,192 토큰에서 4배 확장. rope_theta=1,000,000(일반 모델의 100배)
Hidden Size2,560 (config.json: "hidden_size": 2560) — intermediate_size: 7,168 (≈ 2.8배 확장)
Layers30 (config.json: "num_layers": 30)
Attention HeadsGQA 방식 — Q-head 32개 / KV-head 8개 (head_dim: 80). KV 캐시가 MHA 대비 ¼로 절감되어 긴 문서 처리 및 동시 요청 처리에 유리
TokenizerBPE (Byte Pair Encoding) — vocab size 102,400개. 병합 규칙 merges.txt(1.22MB)에 정의. 한국어 형태소·영어 단어 동시 처리에 최적화

05모델 파일 구성

【config.json (1.04 kB)】 모델 구조 전체를 정의하는 핵심 설정 파일. 레이어 수, hidden_size, attention head 수, RoPE 파라미터, vocab_size 등이 담겨 있으며, 모델 로드 시 가장 먼저 읽힙니다. 이 파일 하나로 아키텍처를 완전히 재현할 수 있습니다.

【configuration_exaone.py (9.95 kB)】 커스텀 아키텍처이므로 ExaoneConfig 클래스를 Python으로 직접 정의한 파일. trust_remote_code=True 설정 시 자동 실행됩니다. 보안 환경에서는 이 파일을 사전에 코드 검토 후 로컬에 등록하는 절차가 필요합니다.

【generation_config.json (134 B)】 기본 생성 파라미터(eos_token_id, pad_token_id 등) 설정. model.generate() 호출 시 기본값으로 사용되며 코드에서 max_new_tokens, temperature 등으로 오버라이드 가능합니다.

【model-00001-of-00002.safetensors / model-00002-of-00002.safetensors (합계 약 9.5 GB)】 실제 모델 가중치. 대용량이므로 2개 파일로 분할 저장. Safetensors 형식은 PyTorch pickle(.pt/.bin)의 악성 코드 실행 위험을 제거하고 메모리 맵핑(mmap)으로 빠른 로드를 지원합니다. bfloat16 추론 시 약 4.8 GB를 사용합니다.

【merges.txt (1.22 MB)】 BPE 토크나이저의 서브워드 병합 규칙표. 문장을 토큰으로 분리할 때 참조합니다. 102,400개 어휘를 커버하는 수만 줄의 규칙이 담겨 있습니다.

【LICENSE (13.8 kB)】 EXAONE AI Model License Agreement 1.1 – NC 전문. 비상업적 사용만 허용되며, 상업적 서비스 또는 경쟁 AI 모델 개발에는 사용할 수 없습니다.

06활용 가능 업무

【공문서 요약·분류】 32K 컨텍스트를 활용해 A4 20~25페이지 분량의 장문 공문, 지침, 고시를 핵심 내용으로 요약하거나 문서 유형을 자동 분류하는 시스템 구축에 활용할 수 있습니다.

【민원 Q&A / 내부 챗봇 (RAG 연계)】 내부 법령·규정·매뉴얼을 벡터 DB에 저장한 뒤 RAG(검색 증강 생성) 파이프라인과 연계하면, 환각 없이 근거 기반 답변을 제공하는 내부 전문 AI를 구축할 수 있습니다. 32K 컨텍스트는 검색 결과를 넉넉히 주입할 수 있어 RAG에 특히 적합합니다.

【정보 추출 (IE)】 계약서, 결재 문서, 연구 보고서에서 날짜·기관명·금액·조항 번호 등 핵심 정보를 구조화된 형태(JSON 등)로 자동 추출합니다.

【보고서 초안 작성】 키워드·개요를 입력하면 행정 보고서, 검토 의견서, 회의록 초안을 생성합니다. 담당자가 검토·수정하는 인간-AI 협업 워크플로우로 업무 효율을 높일 수 있습니다.

【온프레미스 로컬 AI 구동】 Ollama 또는 llama.cpp를 통해 인터넷 연결 없이 내부망(폐쇄망)에서 완전히 독립적으로 운용 가능합니다. 개인정보·기밀 데이터가 포함된 문서도 외부 유출 위험 없이 처리할 수 있습니다.

07한계와 주의사항

【환각(Hallucination) 위험】 모델이 통계적 확률로 텍스트를 생성하므로, 사실과 다른 내용을 그럴듯하게 생성할 수 있습니다. 법령 조항·수치·인명 등 정확성이 중요한 정보는 반드시 원본 문서로 검증해야 합니다.

【최신 정보 미반영】 학습 데이터 수집 이후의 법령 개정, 정책 변경, 최신 사건은 알지 못합니다. 최신 정보가 필요한 업무에는 검색 연계(RAG) 또는 담당자 직접 검토가 필수입니다.

【라이선스 제약 — 비상업적 사용만 허용】 EXAONE AI Model License 1.1 – NC에 따라 대국민 유료 서비스에 직접 적용하거나 경쟁 AI 모델 개발에 활용하는 것은 금지됩니다. 공공기관 내부 업무 효율화 목적은 가능하나, 민간에 서비스로 제공하려면 LG AI Research와 별도 협의가 필요합니다. 도입 전 법무 검토 권장합니다.

【커스텀 아키텍처 의존성】 trust_remote_code=True 설정이 필요하므로, 보안 정책상 원격 코드 실행이 제한된 환경에서는 configuration_exaone.py 코드를 사전 검토 후 로컬에 등록하는 절차가 필요합니다.

【소형 모델의 추론 한계】 2.4B 모델로, 복잡한 다단계 논리 추론이나 전문 도메인 심층 분석에는 더 큰 모델(7B 이상)보다 부족할 수 있습니다. 단순 요약·분류·Q&A에는 충분하나, 정밀한 법률 해석이나 복잡한 수치 계산은 추가 검증이 필요합니다.

【편향 가능성】 학습 데이터에 존재하는 연령·성별·지역 편향이 출력에 반영될 수 있습니다. 의사결정 지원 용도로 사용 시 결과를 맹신하지 말고 전문가 검토를 병행해야 합니다.

08종합 의견

EXAONE-3.5-2.4B-Instruct는 현재 공개된 2~3B급 SLM 중 한국어 처리 능력이 가장 뛰어난 모델입니다. KoMT-Bench 7.24, LogicKor 8.51은 Llama 3.2 3B(각 3.16 / 2.86) 대비 약 2~3배 격차로, 한국어 문서를 주로 다루는 공공·행정·연구기관에서 실용적 가치가 높습니다. Arena-Hard(48.2) 역시 Qwen 2.5 3B(26.4) 대비 약 2배로, 실사용자 선호도 측면에서도 동급 최고 수준입니다.

실무 도입 관점에서는 다음 세 가지 시나리오를 우선적으로 권장합니다: ① 내부 문서 Q&A 시스템(벡터 DB + RAG 파이프라인), ② Ollama 기반 로컬 AI 보조 도구(폐쇄망 내 구동), ③ LoRA 파인튜닝을 통한 기관 특화 모델(내규·전문 용어 적응).

주의할 점은 비상업적 라이선스입니다. 대국민 서비스에 직접 적용하려면 LG AI Research와 별도 협의가 필요하므로, 우선 내부 업무 자동화 POC(개념 검증) 단계부터 시작하는 것이 현실적입니다. 또한 2.4B 소형 모델의 한계상 정밀한 법률 해석이나 복잡한 추론이 필요한 업무에서는 추가 검증이 필수입니다.

결론적으로, EXAONE-3.5-2.4B-Instruct는 한국어 환경에서 로컬 배포가 가능한 최선의 2~3B급 SLM으로, 보안이 중요한 공공기관이나 연구기관의 AI 도입 초기 실험 모델로 가장 적합하다고 판단합니다.
