01모델 선택 이유

Qwen/Qwen3-4B-Instruct-2507은 Hugging Face 기준 약 4.02B 파라미터의 텍스트 생성 모델로, 과제 조건인 1B 이상 7B 이하 범위를 명확하게 충족한다. 모델 카드의 태그와 API 정보 기준으로 text-generation, conversational, transformers, safetensors, license:apache-2.0가 확인되며, 이미지 전용·음성 전용·임베딩 전용 모델이 아니다.

공공기관 업무 관점에서는 단순 Base 모델보다 Instruct 모델이 더 적합하다. 사용자의 지시를 따라 행정문서 요약, 민원 질의응답 초안, 표 기반 자료 정리, 엑셀/PPTX 자동화용 JSON·CSV·파이썬 코드 생성 같은 반복 업무를 수행하기 쉽기 때문이다. 또한 Qwen3 계열은 비교적 최신 모델이며, 긴 컨텍스트와 도구 호출 템플릿을 지원해 문서 처리 자동화 시스템과 결합하기 좋다.

선정 과정에서 Qwen3-VL-2B-Instruct는 규모는 적합하지만 Vision-Language 계열이라 본 과제의 순수 SLM 분석 대상으로는 제외했고, Qwen3-0.6B는 1B 미만이므로 제외했다. Qwen3-4B-Base와 microsoft/phi-2는 조건을 충족하지만, 공공 행정 자동화 목적에서는 지시 수행 능력이 강화된 Instruct 모델인 본 모델이 더 적합하다고 판단했다.

03학습 데이터와 튜닝 방식

모델 카드 공개 범위에서 전체 학습 데이터셋의 상세 목록은 완전히 공개되어 있지 않다. 다만 Qwen3 계열은 일반 지식, 코딩, 수학, 다국어 처리, 장문 문맥 이해를 목표로 사전학습 및 후처리 학습이 적용된 계열로 설명된다. 따라서 보고서에서는 “학습 데이터의 세부 출처는 모델 카드 공개 범위 내에서 제한적으로만 확인 가능하다”고 명시하는 것이 정확하다.

Qwen3 공식 기술 자료 기준으로 Qwen3 계열은 대규모 웹 문서, PDF 유사 문서에서 추출한 텍스트, 수학·코딩 합성 데이터, 장문 컨텍스트 데이터를 활용한 것으로 설명된다. 그러나 Qwen/Qwen3-4B-Instruct-2507 개별 모델의 전체 학습 데이터셋 명칭, 산업별 데이터 포함 비율, 한국어 데이터 비율, SFT·RLHF·DPO 세부 적용 여부는 모델 카드에 완전히 공개되어 있지 않으므로, 공개 범위 내에서는 제한적으로만 확인 가능하다.

선택 모델은 Instruct 버전이므로 사용자의 명령을 따르도록 후처리 학습된 모델이다. Hugging Face의 tokenizer_config.json에는 system, user, assistant, tool_call, tool_response를 처리하는 채팅 템플릿이 포함되어 있어 대화형 지시 수행과 도구 호출 기반 자동화에 활용할 수 있다.

공공기관 적용 시에는 원 모델을 그대로 사용하는 것보다 내부 보안 문서와 개인정보를 제거한 예시 데이터로 RAG를 구성하거나, 비식별화된 업무 매뉴얼·FAQ·서식 데이터로 별도 평가를 수행하는 방식이 적합하다.

05모델 파일 구성

  • README.md: 모델 개요, 사용 방법, 라이선스, 주의사항 등 모델 카드 본문을 제공한다.
  • LICENSE: Apache 2.0 라이선스 전문을 포함한다.
  • config.json: 아키텍처, hidden size, layer 수, attention head 수, context length, dtype 등 모델 구조 정보를 담는다.
  • generation_config.json: 텍스트 생성 시 기본 샘플링·종료 토큰 등 generation 설정을 담는다.
  • model-00001-of-00003.safetensors ~ model-00003-of-00003.safetensors: 모델 가중치가 3개 shard로 분할 저장되어 있다.
  • model.safetensors.index.json: 분할된 safetensors 가중치 파일과 각 파라미터 텐서의 매핑 정보를 제공한다.
  • tokenizer.json, tokenizer_config.json, vocab.json, merges.txt: 텍스트를 토큰으로 변환하고 채팅 템플릿 및 특수 토큰을 처리하는 토크나이저 관련 파일이다.

06활용 가능 업무

방위사업청과 같은 공공기관에서는 반복적인 문서 검토, 요약, 질의응답, 자료 정리 업무가 많다. 이 모델은 자체적으로 엑셀이나 PPTX 파일을 직접 편집하는 프로그램은 아니지만, 자동화 도구와 결합하면 실무에 필요한 중간 산출물을 생성할 수 있다.

  • 행정문서 요약: 사업공고, 제안요청서, 회의록, 지침, 내부 보고자료를 핵심 쟁점·일정·담당부서·조치사항 중심으로 요약한다.
  • 민원·질의응답 초안: 공개 가능한 규정과 FAQ를 RAG로 연결하여 민원 답변 초안이나 내부 상담 답변 초안을 생성한다.
  • 엑셀 업무 자동화: 표 데이터를 JSON/CSV 구조로 정리하거나, pandas, openpyxl 코드를 생성해 반복 집계·분류·검증 작업을 보조한다.
  • PPTX 보고서 초안: 사업 진행 현황, 위험요인, 일정, 예산, 조치계획을 슬라이드 목차와 bullet 초안으로 변환하고, python-pptx 자동 생성 스크립트 작성에 활용한다.
  • 계약·조달 문서 검토 보조: 반복되는 용어, 누락 항목, 일정 불일치, 평가 기준 표현을 검토하는 1차 점검 도구로 활용한다.
  • 업무 매뉴얼 검색: 내부 지침과 규정 문서를 벡터 검색 또는 키워드 검색과 결합하여 담당자가 필요한 조항을 빠르게 찾도록 지원한다.

한국어 전용 벤치마크 근거만 보면 EXAONE 계열이 더 명확하다. 특히 EXAONE-4.0-1.2B는 KMMLU-Pro, KMMLU-Redux, KSM을 공개하고, EXAONE-3.5-2.4B-Instruct는 KoMT-Bench와 LogicKor를 공개한다. 따라서 “한국어 모델임을 강조하는 보고서”라면 EXAONE 선택도 설득력이 있다.

그러나 방위사업청과 같은 한국 공공기관의 반복 행정업무 자동화는 단순 한국어 대화보다 장문 문서 처리, 지시 수행, RAG 검색 결과 기반 답변, 엑셀/PPTX 자동화를 위한 구조화 출력, 내부 도구 호출, 라이선스 유연성이 중요하다. 이 기준에서는 Qwen3-4B-Instruct-2507이 가장 균형이 좋다.

Qwen3의 약점은 한국어 전용 평가 수치가 부족하다는 점이다. 또한 Qwen3 계열 일부 사용 후기에서는 언어 혼합, 반복, 추론 루프 같은 문제가 언급되므로 한국어 행정문서 적용 시 낮은 temperature, 명확한 출력 형식, RAG 근거 제시, 담당자 검토 절차가 필요하다. 다만 Qwen3-4B-Instruct-2507 자체에 대해 한국어 성능이 명확히 부정적이라는 신뢰도 높은 공식 자료는 확인되지 않았다.

최종 판단: 한국어 전용 성능 근거는 EXAONE이 강하지만, 한국 공공기관의 실제 업무 자동화까지 고려한 최종 모델은 Qwen/Qwen3-4B-Instruct-2507로 선정한다.

07한계와 주의사항

  • 환각 가능성: 모델은 실제 법령, 규정, 계약 조건을 잘못 생성할 수 있다. 최종 판단은 반드시 담당자가 원문 문서와 대조해야 한다.
  • 보안 이슈: 방위사업 관련 문서는 민감정보와 보안정보가 포함될 수 있으므로, 외부 API 사용 시 원문 입력 금지, 비식별화, 망분리, 접근통제, 로그 관리가 필요하다.
  • 한국어 검증 필요: Qwen 계열은 다국어 처리에 강점이 있으나, 기관 내부 용어, 군수·조달·계약 전문용어, 한글 서식 표현은 별도 평가가 필요하다.
  • 파일 편집 한계: LLM은 엑셀·PPTX 파일을 직접 안정적으로 조작하는 도구가 아니다. 실제 파일 생성은 pandas, openpyxl, python-pptx 같은 검증 가능한 프로그램과 연계해야 한다.
  • 학습 데이터 불투명성: 모델 카드만으로 전체 학습 데이터의 세부 출처를 완전히 확인하기 어렵다. 공공기관 도입 전 저작권, 개인정보, 보안 적합성 검토가 필요하다.
  • 책임성: 자동 생성된 답변을 대민 답변이나 공식 보고서에 그대로 사용하면 안 되며, 검토자 승인 절차와 변경 이력 관리가 필요하다.

08종합 의견

Qwen/Qwen3-4B-Instruct-2507은 과제의 모델 규모 조건을 명확히 충족하면서도, Instruct 모델이라는 점에서 공공기관의 반복 행정업무 자동화 주제로 분석하기 적합하다. 7B를 초과하지 않는 4B급 모델이므로 로컬 또는 기관 내부 서버에서 POC를 수행하기에도 상대적으로 부담이 낮다.

방위사업청 업무에 적용한다면 “모델 단독 사용”보다는 보안 검토를 거친 내부 문서 검색 시스템, 파일 처리 자동화 코드, 승인 워크플로와 결합하는 방식이 바람직하다. 예를 들어 사용자가 한글 문서나 엑셀 자료를 업로드하면, 시스템이 텍스트와 표를 추출하고, 모델이 요약·분류·검증 규칙을 제안한 뒤, 최종 파일 생성은 검증 가능한 코드가 수행하는 구조가 적합하다.

도입 전에는 한국어 행정문서 요약 정확도, 표 데이터 변환 정확도, 민감정보 노출 위험, 답변 근거 제시 능력, 내부 규정 질의응답 정확도에 대한 별도 평가셋을 만들고, 담당자 검토가 포함된 제한적 POC를 먼저 수행하는 것이 필요하다.
