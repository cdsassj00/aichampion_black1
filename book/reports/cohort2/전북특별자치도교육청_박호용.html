01모델 선택 이유

- EXAONE-3.5-2.4B-Instruct는 LG AI Research가 공개한 한국어·영어 이중언어 instruction-tuned SLM으로, 공공기관에서 많이 다루는 한국어 문서 처리와 질의응답 업무를 검토하기에 적합하다고 판단하였다.
- 모델 규모가 2.4B급으로 과제 조건인 1B 이상 7B 이하 범위에 들어가며, 32K 토큰 장문 컨텍스트를 지원하여 긴 계획서, 지침, 회의자료, 보고서, 매뉴얼 기반 RAG 업무에 적용 가능성을 검토하기 좋다.
- 한국어 벤치마크와 장문 이해 평가 결과가 함께 제시되어 있어 단순 영어권 모델보다 교육행정·민원·내부 문서 분석 관점에서 설명할 근거가 비교적 명확하다.
- 예시 모델을 그대로 사용하는 대신 국내 연구기관이 공개한 한국어 지원 SLM을 선정하여, 전북교육청의 생성형 AI 기반 업무지원·문서 분석·교육데이터 활용 시나리오와 연결해 분석하였다.

03학습 데이터와 튜닝 방식

- 사전학습 데이터: 기술보고서 기준 EXAONE 3.5 2.4B 모델은 약 6.5T 토큰 규모의 학습 코퍼스를 사용한 것으로 제시되어 있다. 다만 개별 원천 데이터셋의 상세 목록은 공개 범위 내에서 모두 확인되지는 않는다.
- 데이터 구성 방식: 1단계 사전학습에서는 일반 도메인 성능 향상을 위해 다양한 출처에서 수집·정제한 대규모 코퍼스를 사용하고, 2단계 사전학습에서는 평가를 통해 보강이 필요한 영역과 장문 이해 능력 향상에 초점을 둔 것으로 설명되어 있다.
- 언어 특성: 기술보고서상 어휘 구성은 한국어와 영어가 대략 50:50 비율로 구성된 것으로 제시되어, 한국어 행정문서 처리 가능성을 검토할 수 있다.
- 튜닝 방식: 모델명과 모델 카드상 Instruct 모델이며, 기술보고서에서는 사전학습 이후 Supervised Fine-Tuning(SFT)과 Preference Optimization을 수행했다고 설명한다. Preference Optimization에는 DPO, SimPO와 같은 직접 선호 최적화 계열 방법이 언급되어 있다.
- LoRA·QLoRA 적용 여부: 공개된 모델 카드와 기술보고서 범위에서는 LoRA 또는 QLoRA 방식으로 학습되었다는 내용은 확인되지 않는다.
- 데이터 준수: LG AI Research는 데이터 수집, 모델 학습, 정보 제공 과정에서 AI Compliance 검토를 수행한다고 설명한다. 다만 공공기관 업무 적용 시에는 별도의 개인정보·보안 검토가 필요하다.

05모델 파일 구성

- README.md: 모델 소개, 사용 방법, 성능 평가, 한계, 라이선스, 인용 정보가 정리된 모델 카드 역할을 한다.
- config.json: 모델 아키텍처와 실행 설정을 담는 핵심 파일이다. hidden_size, num_layers, num_attention_heads, num_key_value_heads, max_position_embeddings, vocab_size 등 모델 구조 분석에 필요한 값이 포함된다.
- tokenizer.json / tokenizer_config.json / vocab.json / merges.txt: 입력 문장을 토큰 단위로 나누고 다시 복원하기 위한 토크나이저 관련 파일이다. 한국어와 영어 문장을 모델이 처리할 수 있는 숫자 ID 시퀀스로 변환하는 역할을 한다.
- model-00001-of-00002.safetensors, model-00002-of-00002.safetensors: 실제 학습된 가중치가 저장된 파일이다. safetensors 형식은 모델 파라미터를 안전하고 효율적으로 저장하는 포맷이며, 이 모델은 가중치가 2개 shard로 나뉘어 제공된다.
- model.safetensors.index.json: 여러 개로 분할된 safetensors 파일 중 어떤 파라미터가 어느 shard에 들어 있는지 알려주는 색인 파일이다.
- generation_config.json: BOS, EOS, PAD 토큰 등 텍스트 생성 시 기본적으로 참조하는 생성 관련 설정을 담고 있다.
- configuration_exaone.py / modeling_exaone.py: EXAONE 전용 설정과 모델 구조를 Transformers에서 불러오기 위한 custom code 파일이다. 따라서 사용 시 trust_remote_code=True 옵션이 필요할 수 있다.

06활용 가능 업무

- 내부 문서 질의응답: 교육청 지침, 업무편람, 회의자료, 계획서, 매뉴얼을 RAG로 연결하여 “해당 업무의 절차가 무엇인지”, “근거 조항이 어디인지”를 찾아주는 내부 업무 챗봇에 활용할 수 있다.
- 공문서·보고서 초안 작성 보조: 긴 참고자료를 바탕으로 추진 배경, 주요 내용, 기대효과, 유의사항 등을 요약하고 개조식 초안을 작성하는 보조 모델로 검토할 수 있다.
- 민원·질의 대응 지원: 공개 가능한 규정, 안내문, FAQ를 기반으로 교직원 또는 민원인의 반복 질의에 대한 초안 답변을 생성하는 데 활용할 수 있다. 단, 최종 답변은 담당자가 검토해야 한다.
- 교육데이터 허브 활용: 데이터 자료실, 데이터 도구실, 데이터 분석실의 게시글·첨부자료를 검색하고 요약하여 사용자가 필요한 자료를 빠르게 찾도록 지원할 수 있다.
- 교육행정데이터통합관리시스템 연계: 데이터 항목 설명, 분석 결과 해석, 지표 정의 안내 등 데이터 기반 행정업무 보조 기능에 활용 가능성이 있다.
- 장문 문서 분석: 32K 토큰 컨텍스트를 활용하면 여러 쪽의 계획서나 지침을 한 번에 넣고 핵심 쟁점, 담당 부서별 조치사항, 검토 필요사항을 추출하는 업무에 적용할 수 있다.
- 온프레미스·폐쇄망 검토: 2.4B급 모델은 상대적으로 작기 때문에 GPU 자원이 제한된 내부망 실험 환경에서 PoC를 수행하기에 대형 모델보다 부담이 작다.

07한계와 주의사항

- 정확성 한계: 모델은 확률적으로 문장을 생성하므로 실제 규정·법령·지침과 다른 내용을 그럴듯하게 생성할 수 있다. 공문, 민원, 감사 대응 등 책임 있는 업무에는 반드시 근거 문서 확인 절차가 필요하다.
- 최신성 한계: 모델 자체는 학습 시점 이후 정보를 자동으로 알 수 없으므로, 최신 지침·조직개편·법령 개정 사항은 RAG 검색 또는 별도 데이터 갱신이 필요하다.
- 한국어 지원 수준: 한국어·영어 이중언어 모델이지만, 전북교육청 내부 행정 용어, K-에듀파인·나이스·교데통 등 특정 시스템 용어는 별도 프롬프트 설계와 내부 문서 기반 보강이 필요하다.
- 라이선스 유의: EXAONE AI Model License Agreement 1.1 - NC 조건이 적용되므로, 연구·검토 목적을 넘어 실서비스 또는 상업적 성격의 운영에 활용할 경우 라이선스 검토와 기관 차원의 법무·보안 검토가 필요하다.
- 개인정보·보안: 학생, 교직원, 민원인 개인정보가 포함된 자료를 외부 환경에 입력해서는 안 되며, 내부망 적용 시에도 접근권한, 로그 저장, 비식별 처리, 민감정보 차단 절차가 필요하다.
- 운영 부담: 모델을 직접 운영할 경우 GPU 서버, 모델 업데이트, 장애 대응, 프롬프트 관리, RAG 데이터 품질관리, 사용자 교육 등 지속적인 운영체계가 필요하다.
- 평가 필요: 실제 교육행정 문서에서 요약 정확도, 근거 인용 정확도, 한국어 답변 품질, 응답 속도, 보안성을 별도 검증한 뒤 단계적으로 적용해야 한다.

08종합 의견

- EXAONE-3.5-2.4B-Instruct는 예시 모델을 그대로 사용하는 대신, 국내 연구기관이 공개한 한국어·영어 지원 SLM을 분석했다는 점에서 과제 취지에 잘 맞는다.
- 모델 규모가 2.4B급이라 과제 조건에 적합하며, 32K 토큰 장문 처리와 instruction-tuned 특성을 갖추고 있어 문서 요약, 내부자료 질의응답, RAG 기반 업무도우미, 보고서 초안 작성 등 교육행정 업무와 연결해 설명하기 좋다.
- 다만 라이선스가 NC 조건을 포함하고, 학습 데이터의 세부 원천 목록이 모두 공개된 것은 아니므로 실제 조직 도입보다는 내부 PoC, 연구, 교육용 실습, 폐쇄망 적용 가능성 검토 단계에 우선 활용하는 것이 적절하다.
- 전북교육청 업무 적용 시에는 ① 공개 가능 자료 기반 RAG 실험, ② 내부 문서 요약 정확도 평가, ③ 개인정보·보안 필터링, ④ 담당자 검토 절차, ⑤ 라이선스 검토를 거친 뒤 제한된 범위에서 단계적으로 검토하는 방안이 적절하다.
- 종합적으로 이 모델은 “작지만 한국어 행정문서 실험에 쓸 수 있는 SLM 후보”로 평가할 수 있으며, AI데통이와 같은 매뉴얼 챗봇, 교육데이터 허브 검색 보조, 공문서 초안 작성 지원 등 업무지원형 AI 서비스의 기초 모델 후보로 검토할 수 있다.
