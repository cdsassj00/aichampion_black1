01모델 선택 이유

본 보고서에서는 성남시청 내부 행정업무에서 보안 걱정 없이 활용할 수 있는 내부 AI 가능성을 중심으로 모델을 선정하였다. 현재 시 내부에서 API를 호출하여 사용하는 AI 서비스는 존재하지만, 직원들이 민원·공문·회의록·내부 지침과 같은 자료를 다룰 때에는 외부 전송과 보안에 대한 우려가 남을 수 있다. 따라서 장기적으로는 내부망 또는 통제된 환경에서 사용할 수 있는 SLM을 검토할 필요가 있다고 판단하였다.

다만 성남시 정보통신과에서 관리하는 GPU 사양을 정확히 알지 못하는 관계로, 7B급 모델이나 Gemma 계열처럼 상대적으로 무거운 로컬 SLM을 바로 도입하는 것은 부담이 될 수 있다고 보았다. Phi-3.5-mini-instruct는 3.8B 규모로 과제의 1B~7B 조건을 충족하면서도 128K token context length를 지원한다. 이 때문에 공공행정 문서처럼 길고 맥락이 많은 자료를 다루는 데 유리하며, 성남시청 내부 AI의 후보 모델로 검토할 가치가 있다고 판단하였다.

03학습 데이터와 튜닝 방식

모델 카드는 Phi-3.5-mini-instruct가 총 3.4조 토큰 규모의 데이터로 학습되었다고 설명한다. 학습 데이터에는 품질 필터링된 공개 문서, 고품질 교육 데이터, 코드, 수학·코딩·상식 추론을 위한 합성 textbook-like 데이터, 그리고 지시 따르기·진실성·정직성·도움성을 반영한 chat format supervised data가 포함된다. 공개 데이터 기준 컷오프는 2023년 10월로 제시되어 있다.

튜닝 방식 측면에서 이 모델은 단순 Base 모델이 아니라 instruction-tuned 모델이다. 모델 카드는 supervised fine-tuning, proximal policy optimization, direct preference optimization을 포함한 강화 과정을 거쳤다고 설명한다. 따라서 사용자 명령을 따르고, 더 안전하고 도움이 되는 답변을 생성하도록 후처리된 모델로 볼 수 있다. LoRA 또는 QLoRA는 모델 카드상 핵심 튜닝 방식으로 명시되지는 않지만, 향후 성남시 내부 문서나 민원 표현에 맞춘 별도 경량 파인튜닝을 검토할 때 활용 가능한 방식이다.

05모델 파일 구성

Hugging Face 저장소의 파일 구성은 모델을 실제로 실행하기 위한 구성요소로 볼 수 있다. config.json은 모델의 구조를 정의하며, hidden size, layer 수, attention head 수, context length 등을 확인할 수 있다. tokenizer.json과 tokenizer.model은 입력 문장을 모델이 처리할 수 있는 토큰으로 변환하는 역할을 한다.

model-00001-of-00002.safetensors, model-00002-of-00002.safetensors 파일은 실제 학습된 가중치가 저장된 파일이며, model.safetensors.index.json은 분할된 가중치 파일을 연결하는 인덱스 역할을 한다. generation_config.json은 답변 생성 시 필요한 eos token, pad token 등 기본 설정을 담는다. modeling_phi3.py와 configuration_phi3.py는 Transformers 환경에서 Phi3 모델 구조와 설정을 해석하기 위한 코드 파일이다.

06활용 가능 업무

성남시청 내부 업무에서는 공문, 보고서, 회의록, 민원 처리 매뉴얼, 내부 지침을 요약하거나 질의응답하는 용도로 활용할 수 있다. 특히 128K 컨텍스트를 지원하므로 여러 쪽에 걸친 회의자료나 지침 문서를 한 번에 넣고 핵심 내용, 결정사항, 담당 부서, 처리 기한을 추출하는 데 유리하다.

상수도, 도로, 복지, 세무, 교통 등 민원 업무에서는 민원 내용을 유형별로 분류하고, 내부 매뉴얼·조례·FAQ를 검색한 뒤 답변 초안을 작성하는 RAG 시스템의 생성 모델로 사용할 수 있다. 현재 API 호출 기반 AI 서비스는 존재하더라도, 직원 입장에서는 보안상 외부 전송이 부담스러울 수 있다. 따라서 내부 문서 검색과 결합한 내부 AI 형태로 운영하면 보안 우려를 줄이면서 행정 문서 요약, 민원 답변 초안 생성, 보고서 초안 작성, 반복 질의응답 지원에 활용할 수 있다.

07한계와 주의사항

Phi-3.5-mini-instruct는 한국어를 지원하지만, 한국 행정문서와 성남시 내부 업무 표현에 특화된 모델은 아니다. 또한 공개 데이터 기준 컷오프가 2023년 10월이므로 최신 법령, 조례, 예산, 지원사업 정보는 모델 자체 지식에 의존해서는 안 된다. 행정 업무에 적용할 경우 반드시 RAG를 통해 최신 내부 문서와 공식 근거를 연결해야 하며, 최종 답변은 담당자가 검토해야 한다.

또한 정보통신과에서 관리하는 실제 GPU 사양을 모르는 상태에서는 7B급 모델이나 Gemma급 로컬 SLM을 바로 도입하기 어렵다. GPU 메모리, 동시 사용자 수, 응답 속도, 내부망 구성, 개인정보 비식별화 정책을 먼저 확인해야 한다. 특히 민원 데이터에는 성명, 연락처, 주소, 민감한 사정이 포함될 수 있으므로 내부 AI라 하더라도 권한 관리와 로그 관리가 필요하다.

08종합 의견

성남시청 소속 사진우의 관점에서 볼 때, Phi-3.5-mini-instruct는 현재 단계에서 내부 AI 후보 모델로 검토하기에 합리적이라고 판단된다. 7B급 또는 Gemma급 모델은 성능 면에서 매력적일 수 있으나, 정보통신과 보유 GPU 사양을 정확히 모르는 상황에서는 운영 부담이 클 수 있다. 반면 이 모델은 3.8B 규모로 상대적으로 가볍고, 128K 장문 컨텍스트를 지원하므로 행정 문서 분석, 민원 질의응답, 내부 지침 기반 RAG에 적합하다.

다만 최종 도입 모델을 지금 확정하기보다는, 추후 정보통신과에서 보유한 GPU 사양을 확인하고 SLM을 기 구축한 부산시 사하구청의 적용 사례를 분석한 뒤 모델을 재설정하는 것이 합당하다고 사료된다. 즉, 본 모델은 현재 과제와 초기 검토 단계에서는 적합하지만, 실제 구축 단계에서는 하드웨어, 보안정책, 내부 문서 규모, 동시 접속 수, 사례 분석 결과를 반영해 Phi-3.5-mini-instruct, 7B급 모델, 또는 다른 경량 모델 중 최종 후보를 다시 비교해야 한다.
