01모델 선택 이유

1 상업적 활용 가능

Qwen2.5 계열은 공개 가중치(Open Weight) 모델로 제공되며, 자체 서버 및 온프레미스 환경에서 실행 가능하다. 별도의 API 사용료 없이 로컬 GPU 서버에 직접 배포 가능하므로 조직 내부 폐쇄망 운영에 적합하다.

2 폐쇄망(On-Premise) 운영 가능

모델 자체를 다운로드하여 내부망 GPU 서버에서 구동할 수 있다. 따라서 인터넷 연결이 차단된 환경에서도 사용 가능하며, 내부 문서 기반 RAG(Retrieval-Augmented Generation) 시스템과 연계하여 운영할 수 있다.

3 한국어 지원

Qwen2.5는 한국어를 포함한 29개 이상의 다국어를 지원한다. 한국어 질의응답, 문서요약, 번역, 코드 생성 등 다양한 태스크에서 비교적 우수한 성능을 제공한다.

4 소형 모델 대비 높은 성능

3B 규모 모델임에도 긴 컨텍스트 처리와 구조화된 출력(JSON 생성 등)에 강점을 보인다. 이는 실제 업무 자동화 및 공공기관 문서 처리 업무에 유리하다.

5 확장성

인터넷망이 허용되는 환경에서는 외부 API 연계(OpenAPI, 사내 API Gateway 등)도 가능하며, LangChain·LlamaIndex·vLLM 등 주요 프레임워크와의 호환성이 우수하다.

03학습 데이터와 튜닝 방식

1 학습 데이터

Qwen2.5는 약 18조(Token 기준) 규모의 대규모 데이터를 활용하여 사전학습(Pretraining)되었다.

포함 데이터 유형은 웹 문서(Web Corpus), 코드 데이터(Code Repository), 수학데이터(Math Corpus), 다국어 데이터(Multiingual Corpus), 문서 및 표(Table) 데이터, 지식 기반 데이터

한국어 데이터 역시 포함되어 있으며, 영어 중심 모델 대비 한국어 처리 성능이 우수한 편이다.

다만 학습 데이터 전체가 완전 공개된 것은 아니며, 일부 데이터는 정제(Curation) 및 필터링 과정을 거친 비공개 데이터셋이 포함된 것으로 알려져 있다.

2 튜닝(Fine-Tuning) 방식

Qwen2.5-Instruct 모델은 다음 단계를 거쳐 튜닝되었다.

 (1) Supervised Fine-Tuning(SFT)

사람이 작성한 질의응답 데이터를 활용하여 지시 수행 능력(Instruct Capability)을 향상시켰다.

 (2) Reinforcement Learning(RLHF 계열)

사람 선호 기반 강화학습을 통해 응답 품질 및 안정성을 개선하였다.

 (3) Alignment Tuning

유해성 감소 및 정책 정렬을 위한 후처리 튜닝이 수행되었다.

 (4) Structured Output Optimization

JSON·표·코드 생성 등 구조화된 출력 품질 개선 튜닝이 포함되었다.

07한계와 주의사항

1 환각(Hallucination) 가능성

사실과 다른 내용을 생성할 가능성이 존재한다.
특히, 법률, 보안 정책, 행정 규정, 숫자 기반 데이터 등에서는 검즐 절차가 반드시 필요하다.

2 한국어 특화 모델 대비 한계

한국어 성능이 우수하지만 완전한 한국어 특화 모델(EXAONE, HyperCLOVA 계열 등) 대비 자연스러움이나 전문 용어 처리에서 부족할 수 있다.

8.3 라이선스 검토 필요

Qwen 계열은 Apache 2.0이 아닌 Qwen Research License를 사용한다. 
따라서, 상업적 서비스 배포, SaaS 서비스화, 재판매 등의 경우 법무 검토가 필요하다. 다만, 일반적인 내부 업무 시스템 및 연구 활용은 현실적으로 많이 사용되고 있다.

4. GPU 자원 요구

3B 모델이라도 실서비스 수준에서는 GPU 자원이 필요하다. 
권장 환경: NVIDA RTX 4090, A100, L40S, H100 등의 GPU 사용이 권장

8.5 보안 이슈

외부 인터넷 연결 시 다음 위험이 존재한다.
 - 프롬프트 인젝션, 데이터 유출 악성 문서 입력, API 호출 보안 문제

따라서, 입려 필터링, 출력 검증, 망분리, API Gateway 보안 정책 등이 필요하다.

08종합 의견

Qwen2.5-3B-Instruct는 현재 공개형 SLM 중에서 성능·운영 효율·다국어 지원의 균형이 우수한 모델이다.
특히 공공기관 및 기업의 내부 지식검색, 문서 자동화, RAG 기반 챗봇 구축 목적에 적합하며, 폐쇄망 운영도 가능하다는 점에서 실제 업무 적용 가능성이 높다.

다만 실제 운영 환경에서는 보안 검증, 라이선스 검토, 환각 검증, 한국어 성능 테스트를 반드시 수행해야 하며, 중요 업무에서는 Human-in-the-Loop 방식의 검수 체계를 함께 운영하는 것이 바람직하다.
